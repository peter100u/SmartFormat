# Mova 许可证披露方案

## 目标

Mova 需要在产品内清楚说明媒体处理能力所依赖的开源组件、构建信息、许可证文本和源码获取方式。这个页面的目标不是把普通用户吓退，而是建立透明感，并为 FFmpegKit、FFmpeg 及第三方媒体库的许可证合规留出明确位置。

本文档用于指导 App 内「许可证」页面、设置页入口、后续商业化说明和法务审阅。

## 产品入口

设置页在「隐私与法律」分组中提供一个胶囊 cell：

```text
许可证
开源组件、FFmpegKit 和第三方许可说明
```

点击后进入二级页面：

```text
许可证
```

## 页面定位

「许可证」页面应覆盖以下信息：

- Mova 使用的开源组件说明。
- FFmpegKit Extended base 小包构建说明。
- FFmpeg、FFmpegKit 及第三方媒体库与 Mova 的关系说明。
- 当前构建的许可证状态。
- 第三方许可证文本入口。
- 源码获取方式。
- Mova 后续商业化能力与开源组件之间的边界。

## 页面首屏文案

建议使用以下文案作为页面顶部说明：

```text
Mova 当前开发阶段使用 FFmpegKit Extended 的 base 构建版本提供本地媒体处理能力。

FFmpegKit、FFmpeg 及相关第三方媒体库是独立开源项目，版权归各自贡献者所有。Mova 与 FFmpeg、FFmpegKit 及其贡献者没有从属、赞助或背书关系。

Mova 后续如果推出高级版，收费对象应是本应用提供的多文件效率、高级预设、自动化工作流和持续维护服务，并不代表购买 FFmpeg、FFmpegKit 或获得其额外商业授权。
```

## 构建信息表达

当前开发阶段以 `ffmpeg_kit_extended_flutter` 的 `base` 小包构建为基础，目标配置为 `type: "base"`, `gpl: false`, `small: true`。页面中应展示实际发布包的构建信息，而不是只写笼统说明。

建议结构：

```text
构建信息
构建组件：FFmpegKit Extended
构建类型：base
GPL 组件：按实际构建配置显示 是/否
平台：iOS / Android
版本：显示实际集成版本号
```

按当前目标配置，建议表达为：

```text
本应用当前开发阶段使用 FFmpegKit Extended 的 base 构建版本。该构建以基础 FFmpeg 能力为主，具体许可证取决于实际启用的组件。我们会在本页面列出当前版本使用的组件、许可证文本和源码获取方式。
```

上线前必须用实际构建产物确认 GPL 状态，不能只按产品规划填写。

## 组件列表

许可证页面应包含一个「使用的开源组件」分组。首批至少包含：

```text
FFmpegKit Extended
用途：连接 Flutter App 与 FFmpeg 媒体处理能力
构建版本：base（small package）
许可证：当前目标配置按非 GPL 构建处理，页面中仍应展示实际集成版本对应的 LGPLv3 / 第三方许可证信息
项目地址：https://pub.dev/packages/ffmpeg_kit_extended_flutter

FFmpeg
用途：本地音视频格式转换、压缩、提取和文件预检分析
许可证：按当前非 GPL 目标配置展示实际启用组件对应的 LGPL / 第三方许可证信息
官网：https://ffmpeg.org
许可证说明：https://ffmpeg.org/legal.html
```

后续如果启用具体第三方库，应继续列出，例如 x264、x265、libvpx、libwebp、opus、vorbis、dav1d 等。每个条目至少包含：

- 组件名称。
- 用途。
- 许可证。
- 项目地址。
- 源码或许可证地址。

## 许可证文本

页面中应提供「许可证文本」分组：

```text
许可证文本
FFmpegKit 许可证
FFmpeg 许可证
第三方组件许可证
```

许可证文本可以使用 App 内页面展示，也可以打开本地打包的文本文件。不要只提供外链，因为用户在离线状态下也应该能查看随 App 分发的许可证信息。

## 源码获取

页面中应提供「源码获取」分组：

```text
源码获取
查看或获取相关开源组件源码
```

源码获取说明应包含：

- 当前 App 版本对应的 FFmpegKit / FFmpeg 构建信息。
- 对应源码包、构建脚本或上游仓库链接。
- 如许可证要求提供完整对应源码，应提供可访问的下载地址或获取方式。
- 如果 App 使用自建 FFmpeg 二进制，应保存对应源码、补丁、构建参数和构建脚本。

建议文案：

```text
我们按照相关开源许可证要求提供当前版本使用的开源组件信息、许可证文本和源码获取方式。你可以通过本页面列出的链接查看或获取相关源码。
```

## 商业化边界

未来如果加入付费页、高级版页面或商业化说明，必须避免把收费描述成出售 FFmpeg 或 FFmpegKit。

推荐表达：

```text
Mova 高级版解锁的是本应用提供的多文件效率、高级预设和自动化工作流。基础单文件转换永久免费。
```

禁止或避免使用：

```text
购买 FFmpeg Pro
FFmpeg 高级版
我们提供 FFmpeg 商业授权
独家 FFmpeg 转码能力
```

## 设置页信息架构

设置页「隐私与法律」分组建议保持为：

```text
隐私与法律
- 隐私政策
- 用户协议
- 许可证
```

「许可证」比「开源许可证」更适合当前产品，因为该页面不仅包含开源许可证，也包含构建信息、源码获取、第三方组件和商业付费边界说明。

## 合规检查清单

每次发布前应检查：

- 当前构建类型是否与页面展示一致。
- 是否启用了 GPL 组件。
- 是否包含 `nonfree` 组件；如包含，必须重新评估是否可分发。
- App 内是否包含所有必要许可证文本。
- 是否提供源码获取方式。
- 是否保留 FFmpeg、FFmpegKit 和第三方库的版权声明。
- 用户协议是否没有禁止许可证允许的调试、修改或逆向工程行为。
- 付费页或高级版说明没有把收费描述成 FFmpeg 或 FFmpegKit 的商业授权。

## 备注

本文档是产品和实现层面的许可证披露方案，不构成法律意见。正式上架前，尤其在调整 GPL 状态、使用自建二进制或进入海外应用商店分发时，应进行一次开源合规审阅。
